# 내가 아는 것을 AI에게 잘 물어보는 법

이미 알고 있는 영역에서 AI를 활용하는 건 가장 효과가 큰 사용법이다. 답이 맞는지 판단할 수 있기 때문에 hallucination에 속을 위험이 낮고, AI를 속도 부스터로 쓸 수 있다.

---

## 1. 반복 작업을 위임하라

이미 할 줄 아는 일이지만 시간이 걸리는 작업이 최적의 위임 대상이다.

- 보일러플레이트 코드 생성
- 테스트 케이스 작성
- 데이터 포맷 변환
- 정규표현식 작성

직접 하면 10분, AI에게 시키면 10초. 결과를 검증할 능력이 있으니 안전하다.

---

## 2. 더 나은 방법이 있는지 물어라

자기 방식대로 해결은 할 수 있지만, 최선인지 모를 때 AI를 활용한다.

**예시:**
> 나는 이 문제를 for 루프로 풀었는데, 더 깔끔한 방법이 있어? 내 코드는 이거야: [코드]

자기 코드를 기준으로 보여주면 AI가 비교 가능한 대안을 제시해준다. 이미 알고 있으니 대안의 장단점도 판단할 수 있다.

---

## 3. 설명 상대로 활용하라 (러버덕 디버깅)

알고 있는 내용을 AI에게 설명하듯 정리하면, 빈틈이 드러난다.

- "내가 이해한 바로는 이건데, 맞아?"
- "이 개념을 주니어 개발자에게 설명한다면 어떻게 할까?"
- "내 설명에서 부정확한 부분이 있으면 지적해줘"

---

## 4. 코드 리뷰어로 활용하라

본인이 작성한 코드를 AI에게 리뷰 요청하면 놓친 엣지 케이스나 개선점을 잡아낼 수 있다.

**효과적인 요청:**
> 이 코드 리뷰해줘. 특히 에러 핸들링과 성능 측면에서 봐줘. [코드]

본인이 해당 코드를 이해하고 있으므로, AI의 리뷰가 적절한지 즉시 판단 가능하다.

---

## 5. 기억 보조로 활용하라

알고 있지만 당장 기억나지 않는 것들을 빠르게 꺼내는 용도로 쓴다.

- "Python에서 딕셔너리 컴프리헨션 문법이 뭐였지?"
- "git에서 특정 커밋으로 되돌리는 명령어가 뭐야?"
- "HTTP 상태 코드 429가 뭐였지?"

검색 엔진 대신 AI를 쓰면 맥락에 맞는 답을 바로 얻을 수 있다. 이미 아는 영역이라 답의 정확도도 바로 검증된다.

---

## 요약

| 활용법 | 핵심 |
|--------|------|
| 반복 작업 위임 | 할 줄 알지만 시간 걸리는 일을 맡겨라 |
| 대안 탐색 | 내 방식이 최선인지 물어라 |
| 러버덕 디버깅 | AI에게 설명하며 빈틈을 찾아라 |
| 코드 리뷰 | 놓친 부분을 잡아달라고 요청하라 |
| 기억 보조 | 아는데 기억 안 나는 건 바로 물어라 |
